花指令

讲解

首先什么是花指令，我的理解是不影响程序正常运行下的，进行静态分析混淆、使反编译器报错的嵌入指令；

以下是Genimi的看法：

花指令（Garbage Instructions）是一种在逆向工程中常见的技巧，通常用于混淆代码，增加分析的难度。简单来说，花指令就是一些无实际作用或没有被执行的代码指令。

它们主要有以下几个特点：

  1. 无意义指令: 这些指令本身可能是一些合法的CPU指令，比如 NOP（空操作），XOR EAX, EAX（将EAX寄存器清零），或者一些简单的加减法，但它们对程序的逻辑没有任何影响。
  2. 混淆控制流: 花指令可以与跳转指令结合使用，通过一些复杂的、不必要的跳转来使程序的执行流程看起来很混乱，让逆向工程师难以追踪真正的代码路径。
  3. 防止自动化分析: 许多反汇编器和自动化分析工具在遇到大量的花指令时会出错，或者无法正确识别程序的真实逻辑，从而使分析工作变得更加困难。
  4. 动态生成: 有些高级的混淆技术会动态生成这些花指令，使得每次程序运行时，指令序列都会有所不同，进一步增加了静态分析的难度。

逆向分析时，常见的、要处理的花有2、3这两点特点（动态花还没遇到过），接下来就从常遇见的花的生成来进一步讲解

生成

对于大部分会报错的花，原因都是反编译器在识别流程时，误判断出一条必定执行不了的流程，然后反汇编了

1.永恒跳转

在这种情况下，ida识别出一条zf（零标志位）先不为0，后为0的流程，然后将E8机器码反汇编为call语句

__asm {
    jz start;
    jnz start;
    _emit 0xE8;
}
start:
    printf("1\n");

2.必定跳转

通过必定跳转，然后在两个地址内添加垃圾指令

    __asm {
        xor eax, eax;
        jz s;
        _emit 0x71
        _emit 3
        _emit 0xE0
        _emit 0xED
    s:
    }

3.修改ESP

该方法仅影响ida对于堆栈指针分析，不妨碍反编译

__asm {
    xor eax, eax;
    jz orz;
    add esp, 0x11;
orz:
}
printf("3\n");;

4.jump跳

在jump指令跨越的中间插入垃圾指令，（有问题，jmp对于递归扫描的ida是无干扰的，同时该jump $+5 语句跳转到 0x71 ，此处的干扰是0x71 0x02造成的）

__asm {
    jmp $ + 5
    _emit 0x71
    _emit 2
    _emit 0xE9
    _emit 0xED
}
printf("4\n");

证明：

修改了jmp寻址后，使其可以跳过花指令，这时，花指令就干扰不了反编译了

初步总结：以上的初步花，重点关注永恒跳和必定跳，以及插入垃圾代码，构建汇编语句
