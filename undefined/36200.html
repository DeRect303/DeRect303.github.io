windows代理查看

方法一：通过 Windows 设置（图形界面）

这是最直观和用户友好的方法。

  1. 打开“设置”： 点击屏幕左下角的“开始”菜单，然后点击齿轮图标进入“设置”。
  2. 进入“网络和 Internet”： 在设置窗口中，选择“网络和 Internet”。
  3. 选择“代理”： 在左侧菜单中，找到并点击“代理”。
  4. 查看代理设置：
    • “自动检测设置”： 如果开启，系统会尝试自动检测代理设置（WCAD）。
    • “使用安装脚本”： 如果开启并配置了“设置脚本地址”，则系统正在使用 PAC (Proxy Auto-Configuration) 文件。
    • “手动设置代理”： 如果开启并配置了“代理服务器”地址和端口，则系统正在使用固定代理服务器。

方法二：通过命令行（CMD 或 PowerShell）

您也可以使用命令行工具来查看代理设置。

1. 使用 netsh 命令 (CMD 或 PowerShell)：

这个命令主要显示 WinHTTP 代理设置，这些设置可能被一些应用程序使用（比如一些非浏览器应用）。

打开命令提示符 (CMD) 或 PowerShell，输入：

DOS

  • 如果显示 “直接访问(无代理服务器)”，表示没有配置 WinHTTP 代理。
  • 如果显示 “代理服务器：<地址:端口>”，则会显示代理服务器的地址和端口。
  • 如果显示 “绕过列表：…”，则表示哪些地址不经过代理。

2. 检查环境变量 (CMD 或 PowerShell)：

有些应用程序可能会读取环境变量来获取代理设置，特别是 http_proxy, https_proxy 和 no_proxy。

在 CMD 或 PowerShell 中输入：

DOS

  • 如果这些变量被设置，它们会显示对应的代理地址。如果为空，则表示没有通过环境变量设置代理。

3. 查看注册表 (PowerShell，更高级)：

Windows 的代理设置主要存储在注册表中。这种方法更技术性，但可以获取详细信息。

打开 PowerShell，输入：

PowerShell

  • ProxyEnable: 如果为 1，表示手动代理已启用；如果为 0，表示手动代理未启用。
  • ProxyServer: 手动代理服务器的地址和端口。
  • AutoDetect: 如果为 1，表示自动检测设置已启用。
  • AutoConfigURL: 自动配置脚本（PAC 文件）的 URL。

总结：

对于大多数用户和应用程序而言，通过 Windows 设置界面查看是最直接和完整的。 如果您想通过命令行快速检查，netsh winhttp show proxy 是一个很好的起点。

您想查看 Windows 系统的环境变量，特别是像 HTTP_PROXY 这样的环境变量。环境变量可以存储系统或用户级别的配置信息，供程序使用。

以下是几种查看 Windows 环境变量的方法：

1. 通过系统属性（图形界面）

这是最常见和官方的方法，可以查看所有系统和用户环境变量。

  1. 打开“系统属性”对话框：
    • 按下 Win + R 键打开“运行”对话框。
    • 输入 sysdm.cpl 并按回车。
  2. 导航到“高级”选项卡：
    • 在弹出的“系统属性”窗口中，点击顶部的 “高级” 选项卡。
  3. 点击“环境变量”按钮：
    • 在“高级”选项卡中，点击底部的 “环境变量(N)…” 按钮。
  4. 查看环境变量：
    • 在“环境变量”窗口中，您会看到两个主要部分：
      • “用户变量”： 针对当前登录用户生效的环境变量。
      • “系统变量”： 对所有用户和整个系统生效的环境变量。
    • 您可以在这两个列表中查找 HTTP_PROXY、HTTPS_PROXY 或其他您感兴趣的变量。如果存在，会显示其变量名和对应的值。

2. 通过命令提示符 (CMD)

您可以使用 set 命令在命令提示符中查看当前会话的所有环境变量，或查看特定变量的值。

  1. 打开命令提示符：

    • 按下 Win + R 键，输入 cmd 并按回车。

  2. 查看所有环境变量：

    • 输入 set 并按回车。
    • 这会列出当前命令提示符会话中所有已定义的环境变量及其值。您可以滚动查找 HTTP_PROXY。

  3. 查看特定环境变量的值：

    • 要查看 HTTP_PROXY 的值，输入：

      DOS

    • 同样，可以查看 HTTPS_PROXY：

      DOS

    • 如果变量不存在，echo 命令会直接输出变量名本身，例如 ECHO 处于关闭状态。 或者 %HTTP_PROXY%。

3. 通过 PowerShell

PowerShell 提供了更强大的方式来操作和查看环境变量。

  1. 打开 PowerShell：

    • 按下 Win + R 键，输入 powershell 并按回车。

  2. 查看所有环境变量：

    • 环境变量在 PowerShell 中存储在 Env: 驱动器中。
    • 输入 Get-ChildItem Env: 或 dir Env: 并按回车。
    • 这会列出所有环境变量。

  3. 查看特定环境变量的值：

    • 使用 $env: 前缀来访问环境变量。

    • 要查看 HTTP_PROXY 的值，输入：

      PowerShell

    • 同样，可以查看 HTTPS_PROXY：

      PowerShell

    • 如果变量不存在，则会返回空行。

总结：

  • 图形界面（系统属性） 适用于全面查看和管理所有用户和系统环境变量。
  • CMD 的 echo %VAR_NAME% 或 PowerShell 的 $env:VAR_NAME 适用于快速查看特定环境变量的值。
  • CMD 的 set 或 PowerShell 的 Get-ChildItem Env: 适用于列出当前会话的所有环境变量。

选择最适合您需求的方法即可。通常对于 HTTP_PROXY 这样的代理变量，使用命令行快速检查会比较方便。
