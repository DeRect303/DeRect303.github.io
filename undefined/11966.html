readelf简介

Created2025-07-31|Updated2025-07-31|Windows
|Post Views:

readelf 工具确实可以看地址，特别是与ELF（Executable and Linkable Format）文件中各种节（sections）、段（segments）以及符号（symbols）相关的地址信息。

readelf 的作用：

readelf 是一个用于显示 ELF 格式文件的信息的命令行工具，它是 GNU Binutils 工具集的一部分。ELF 是一种常用的可执行文件、目标代码、共享库（如 .so 文件）和核心转储文件的格式，广泛用于 Linux、Android 等操作系统。

readelf 可以看出哪些地址信息？

  1. 节（Sections）的地址：
    • 节的虚拟地址（Virtual Address, VA）：加载到内存中的起始地址。
    • 节的文件偏移（File Offset）：在文件中的位置。
    • .text（代码段）、.data（已初始化数据段）、.rodata（只读数据段）、.bss（未初始化数据段）等都是常见的节。
  2. 段（Segments）的地址：
    • 程序头（Program Header）中定义的各个段的虚拟地址和文件偏移。段是运行时加载器关注的更高级别的结构，通常一个段会包含一个或多个节。
  3. 符号（Symbols）的地址：
    • 函数、全局变量等符号的地址（Value 字段）。
  4. 重定位（Relocations）的地址：
    • 需要在加载时进行地址修正的位置。
  5. 动态链接信息（Dynamic Linking Information）的地址：
    • 动态符号表、依赖库等地址。

如何使用 readelf 查看地址信息？

您可以使用不同的选项来查看特定类型的地址信息。

1. 查看节头（Section Headers）信息：

这个命令会列出文件中所有节的名称、类型、地址、偏移、大小等详细信息。

Bash

  • 输出示例中的关键列：
    • Addr (Address)：节的虚拟内存地址。
    • Off (Offset)：节在文件中的偏移量。
    • Size：节的大小。
    • Name：节的名称。

2. 查看程序头（Program Headers / Segments）信息：

这个命令会显示 ELF 文件的程序头表，它定义了如何将文件映射到内存中。

Bash

  • 输出示例中的关键列：
    • Offset：段在文件中的偏移量。
    • VirtAddr (Virtual Address)：段在内存中的起始虚拟地址。
    • PhysAddr (Physical Address)：段在物理内存中的起始地址（通常与 VirtAddr 相同，除非是特殊情况）。
    • MemSiz (Memory Size)：段在内存中占据的大小。
    • FileSiz (File Size)：段在文件中占据的大小。

3. 查看符号表（Symbol Table）信息：

这个命令会列出文件中的符号（函数、变量等）及其地址、大小、类型等。

Bash

  • 输出示例中的关键列：
    • Value：符号的地址。对于函数，这是其入口点的地址；对于变量，这是其存储位置的地址。
    • Size：符号的大小（例如，函数代码的大小）。
    • Name：符号的名称。

4. 查看所有信息 (通常用于全面分析)：

Bash
