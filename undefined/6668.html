010editor:

010 Editor 是一款功能强大的文本和十六进制编辑器，主要特点如下：

核心功能

  • 十六进制编辑：这是 010 Editor 最突出的功能。它可以让你直接以十六进制的形式查看、编辑和分析任何文件，包括二进制文件、可执行文件、磁盘镜像等。这对于逆向工程、数据恢复和文件格式分析非常有用。
  • 模板（Templates）：010 Editor 最强大的功能之一。它使用一种名为 Binary Templates 的脚本语言，可以让你为不同的文件格式（如 ZIP、JPEG、MP3 等）创建模板。这些模板能自动解析文件的结构，让你以可读的方式查看和编辑文件中的每一个字段，而无需手动计算偏移量和大小。
  • 文本编辑：除了十六进制编辑，010 Editor 也是一个完整的文本编辑器。它支持语法高亮、代码折叠、查找替换、书签等常见功能，可以用来编辑源代码、脚本文件或任何普通文本。

其他亮点

  • 数据分析：提供了多种工具来分析文件数据，例如直方图（Histogram）、校验和（Checksum）和哈希值（Hash）。
  • 多平台支持：可在 Windows、macOS 和 Linux 上运行。
  • 脚本（Scripts）：支持使用脚本来自动化重复性任务，例如批量处理文件或修改数据。
  • 工作区：可以同时打开并管理多个文件，方便在不同文件之间进行切换和比较。

DIE：

DIE，全称 Detect It Easy，是一款非常实用的文件类型和加密壳检测工具。它主要用于帮助逆向工程师、安全研究人员或普通用户快速识别一个可执行文件（如 .exe、.dll）或任何其他文件的相关信息。

简单来说，它的作用就像是给文件做一次“体检”，告诉你：

  • 文件是什么类型：是 PE（Portable Executable）文件、ELF 文件还是 Mach-O 文件？这是识别可执行文件的基础。
  • 使用了什么编译器或打包器：文件是用 Visual C++、GCC 还是其他编译器编译的？这能帮助你了解开发环境。
  • 是否被加密或加壳：这是 DIE 最核心的功能之一。它能检测文件是否使用了像 UPX、ASPack、Themida 等常见的加壳工具。加壳是一种保护程序的方式，通常会压缩或加密原始代码，使得逆向分析变得更困难。DIE 能识别出这些“外壳”，为后续的脱壳工作提供线索。
  • 使用了什么语言或库：有时，它还能检测到文件使用的编程语言或依赖的特定库。

XDbg

XDBG 简介

XDBG 是一款专为逆向工程和软件漏洞分析设计的动态调试器。它允许你在程序运行时暂停执行、检查内存、修改变量，从而深入理解程序的工作原理。

核心功能与特点

  1. 动态调试：这是所有调试器的核心功能。XDBG 让你可以在程序运行到特定位置（例如断点）时暂停它，然后逐行执行代码，观察程序的内部状态。这对于分析恶意软件或理解复杂程序逻辑至关重要。
  2. 插件化架构：XDBG 最突出的特点之一是其高度模块化的插件系统。这意味着你可以根据自己的需求，通过安装不同的插件来扩展它的功能。例如，可以添加反混淆插件、代码分析插件或针对特定文件格式的解析插件。
  3. 支持多种 CPU 架构：除了常见的 x86 和 x64 架构，XDBG 还支持其他架构的调试，这使得它在分析嵌入式设备、移动应用或游戏等场景时非常灵活。
  4. 脚本化：XDBG 支持使用脚本来自动化一些重复性的调试任务，例如自动设置断点、搜索特定的内存模式或修改程序行为。这对于自动化逆向分析流程非常有帮助。
  5. 内存分析：它提供了强大的内存查看和修改功能，你可以检查堆栈、堆、寄存器等区域的数据，甚至直接在内存中修改程序运行时的行为。

当我们使用xdbg打开程序时，它就是处于运行时，可以开始调试；它比之ida更注重动态调试、dump、Scylla插件修补IAT表、以及对内存的操作

Cyberchef：

CyberChef

CyberChef 是一个功能强大的Web 工具，被昵称为“网络瑞士军刀”。它的主要作用是提供一个统一的界面来执行各种数据处理操作，从简单的编码/解码到复杂的加密/解密和数据分析。它最大的特点是：

  • “食谱”（Recipe）模式：你可以通过拖拽的方式，将不同的操作（称为“食谱”）串联起来，形成一个处理流程。例如，你可以先对一个字符串进行 Base64 解码，然后进行 Gzip 解压缩，最后进行 URL 解码。这大大简化了处理复杂数据时的流程，无需编写代码。
  • 丰富的操作库：它支持数百种操作，包括：
    • 编码/解码：如 Base64、URL 编码、十六进制等。
    • 加密/解密：支持 AES、DES、RSA 等多种现代和古典加密算法。
    • 数据格式化：JSON 美化、XML 转换等。
    • 散列计算：SHA-1、MD5 等。
    • 其他实用工具：文件分析、IP 地址解析、正则表达式匹配等。

因为是一个 Web 工具，你可以在浏览器中直接使用，而且它还支持离线模式，保证了数据的隐私和安全。

一个强大的加解密工具，囊括了许多常见标准加解密，并支持多个模块选择、参数修改等功能，同时可以部署到本地

jadx：

Jadx（Java Android Decompiler eXpress）是一款用于反编译 Android DEX 和 APK 文件的开源工具。简单来说，它能将 Android 应用的机器码（DEX 字节码）转换回可读的 Java 源代码。

  • 核心功能：它的主要任务就是反编译。当一个 Android 应用被编译后，它包含的源代码被转换成了 DEX 文件，人类难以直接阅读。Jadx 的作用就是将这个 DEX 文件中的指令，转换成我们可以理解的 Java 代码，从而帮助开发者或安全研究人员分析应用的行为。
  • GUI 界面：Jadx 提供了一个直观的图形用户界面（GUI）。你只需将 APK 文件拖放到界面中，它就会自动开始反编译，并在左侧的树形结构中展示应用的类、资源等信息。你可以直接点击查看反编译后的 Java 源代码。
  • 主要用途：
    • 逆向工程：分析恶意软件的行为。
    • 安全审计：检查应用是否存在漏洞。
    • 代码学习：研究开源应用是如何实现的。

jadx是反编译apk文件还原出java层逻辑的工具，以 NSS上的[SWPUCTF 2021 新生赛]easyapp 题目做示例

jadx的主要功能就是将apk的dex文件反编译成java层逻辑，定位到源代码-com-MainActivity就是简单安卓题的流程

Jeb

Jeb 是一款专业的反编译和逆向工程工具，主要用于分析 Android 应用（.apk 文件） 和 Java 字节码。它在移动安全研究、恶意软件分析以及软件漏洞挖掘等领域非常流行。

Jeb 最核心的功能就是反编译，它能将编译后的代码（如 Dalvik 字节码或 Java 字节码）转换回可读性很高的源代码。

核心功能与亮点

  • Android 反编译：
    • Dalvik 字节码：Jeb 的一大优势在于它能够高效地将 Android 应用中的 .dex 文件反编译成接近原始的 Java 源代码。它会自动处理很多复杂的代码结构，使得反编译后的代码更易于理解。
    • 强大的分析能力：除了反编译，Jeb 还能进行各种静态分析，例如自动重命名变量和函数、推断数据类型、识别代码结构等，这极大地减轻了人工分析的负担。
    • 交互式分析：它提供了一个图形用户界面（GUI），你可以在其中方便地浏览应用的类、方法和资源。你甚至可以对代码进行交互式修改，例如重命名变量、添加注释等，这些修改会自动反映在整个程序中。
  • Java 反编译：
    • Java 字节码（.class）：Jeb 同样支持将普通的 Java .class 文件反编译成 Java 源代码。这对于分析没有提供源代码的 Java 程序非常有用。
  • 插件系统：
    • 可扩展性：Jeb 拥有一个强大的插件系统，允许用户编写 Python 或 Java 脚本来扩展其功能。这使得它能够自动化一些重复性任务，或者添加新的文件格式解析器。

Jeb具有类ida形视图，将dex字节码与内存地址关系建立汇编视图，也具备反编译为Java语言的功能

pyinstxtractor

PyInstaller Extractor 简介

PyInstaller Extractor（通常简称为 pyinstxtractor.py）是一个专为逆向分析而设计的 Python 脚本。它的主要功能是解压由 PyInstaller 工具打包生成的 EXE 文件。

PyInstaller 是什么？

在介绍 Extractor 之前，我们需要先了解一下 PyInstaller。PyInstaller 是一个流行的 Python 库，可以将 Python 脚本及其依赖项打包成一个独立的、可执行的 EXE 文件。这样，用户即使没有安装 Python 环境，也可以直接运行这个程序。

PyInstaller Extractor 的作用

当一个 Python 脚本被 PyInstaller 打包成 EXE 后，原始的 .py 或 .pyc 文件会被压缩并嵌入到这个 EXE 中。如果没有 Extractor，我们很难直接查看原始的 Python 源代码。

pyinstxtractor.py 这个脚本就是为了解决这个问题而存在的。它的工作原理是：

  1. 解析 EXE 文件的结构：它会找到 EXE 文件中 PyInstaller 打包的数据段。
  2. 解压数据：它会把这个数据段解压出来，还原成一个文件夹。
  3. 提取 .pyc 文件：在这个文件夹中，你会找到被打包的原始 .pyc（Python 编译后的字节码）文件。

ps：

如果你使用 pyinstxtractor，请在 pyinstxtractor.py 找到 # Skip PYZ extraction if not running under the same python version 然后将它下面五行的 return 注释掉（让 return 不生效）。否则，你运行 pyinstxtractor.py 时使用的 Python 小版本号必须与附件相同。或者，你也可以尝试使用更方便的工具，例如 pydumpck 。

pycdc：

pycdc 简介

pycdc 是一款专门用于反编译 Python 字节码（.pyc 文件）的工具。简单来说，它的作用是把 Python 解释器运行的 .pyc 文件，还原成我们人类可以阅读的 .py 源代码。

什么是 Python 字节码？

当你在运行一个 .py 文件时，Python 解释器并不会直接执行它，而是先将其编译成一种中间语言，也就是字节码。这些字节码会被存储在 .pyc 文件中。这样下次再运行同一个文件时，解释器就可以直接加载 .pyc，省去了编译的时间，提高了程序的启动速度。

pycdc 的作用

pycdc 就是针对这些 .pyc 文件设计的。它的核心功能包括：

  • 反编译：将 .pyc 文件中的字节码指令，通过一系列的算法和规则，转换成对应的 Python 源代码。
  • 版本兼容性：它能够支持多种 Python 版本生成的 .pyc 文件，从旧版本到新版本都有一定的支持。
