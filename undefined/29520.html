ESM模块简单介绍

Created2025-08-01|Updated2025-08-01|ESM
|Post Views:

ES 模块 (ECMAScript Modules)，简称 ESM，是 JavaScript 官方的模块系统标准。它为 JavaScript 提供了导入 (import) 和导出 (export) 功能，让你能够将代码拆分成独立的文件（模块），然后在需要时将它们组合起来。

ES 模块的特点

  • 静态结构: ESM 的导入和导出是在代码执行前解析的。这意味着打包工具（如 Webpack）可以在不运行代码的情况下，分析模块依赖关系，从而进行 Tree Shaking（移除未使用的代码）。
  • 异步加载: 在浏览器中，ESM 是异步加载的，不会阻塞 HTML 的解析。
  • 严格模式 (Strict Mode): ES 模块默认在严格模式下运行，这意味着你不能使用一些不安全的语法，比如使用未声明的变量。
  • 导入导出语法: 使用 import 和 export 关键字。

与 CommonJS (CJS) 模块的区别

在 ES 模块出现之前，Node.js 主要使用 CommonJS（CJS）模块系统，它使用 require() 和 module.exports。ESM 和 CJS 有几个关键区别：

  • 语法: ESM 使用 import/export，而 CJS 使用 require/module.exports。
  • 加载方式: ESM 支持静态分析和异步加载，而 CJS 是同步加载的。
  • 值传递: ESM 导入的是一个值的引用，如果原始模块的值改变，导入的值也会同步更新。而 CJS 导入的是一个值的拷贝，原始模块的改变不会影响导入的模块。
  • 运行时环境: ESM 是 JavaScript 的官方标准，可以在浏览器和 Node.js 中使用（需要一些配置或较新版本）。CJS 主要用于 Node.js。

现在，新版本的 Node.js 也已经全面支持 ES 模块，并且鼓励开发者在项目中采用 ES 模块。
